# JOOTOPIA프로젝트에서 내가 한 부분

## 뉴스 감성 분석 및 파인튜닝

### 1. 전체 접근 개요

본 프로젝트에서 내가 맡은 파트의 뉴스 감성 분석은 **정확도보다 대량 처리 효율성과 일관성**을 핵심 목표로 설계되었습니다.

1. **1단계**: 100개 종목 뉴스 기반 키워드 분석을 통한 감성 라벨 생성
2. **2단계**: 금융 도메인 특화 언어모델 파인튜닝
3. **3단계**: 학습된 모델을 활용한 307개 종목 전체 뉴스 감성 분류

---

### 2. 키워드 기반 라벨링

초기에는 대규모 정답 라벨 확보를 위해 Gemini API 활용을 검토했으나,
일일 호출 수 및 토큰 제한으로 인해 **대량 뉴스 처리에 한계**가 있었습니다.

이에 대한 대안으로,

* 100개 종목의 뉴스 데이터를 대상으로
* **호재 / 악재 / 중립 키워드 사전을 정의**
* 키워드 출현 패턴을 기반으로 감성 라벨을 생성

이 방식을 통해 **일관성과 재현성이 있는 학습용 데이터셋**을 확보했습니다.

---

### 3. 베이스 모델 선정 및 파인튜닝 필요성

베이스 모델로는 **금융 도메인 텍스트 처리에 특화된 kakaobank/kf-deberta-base 모델**을 사용했습니다. 선정 이유는 다음과 같습니다.

1. 금융 도메인 특화 데이터셋 (Domain-Specific Corpus)
비교: 일반 BERT나 KoBERT는 위키피디아, 뉴스 등 일반적인 문어체로 학습되어 '유상증자', '액면분할'과 같은 경제 전문 용어 파악에 한계가 있습니다.

강점: 본 모델은 12.3GB 이상의 대규모 한국어 금융 말뭉치로 사전 학습되어, 주식 시장 특유의 문맥과 경제 용어를 왜곡 없이 정확하게 이해합니다.

2. 최신 DeBERTa v3 아키텍처의 성능 (Advanced Architecture)
비교: 1세대 모델인 BERT 계열보다 문장 구조 파악 능력이 뛰어납니다.

강점: Disentangled Attention 구조를 통해 단어의 의미와 위치 관계를 분리하여 학습하므로, 주어와 목적어에 따라 호재/악재가 급변하는 주식 뉴스의 복잡한 문장 구조를 훨씬 정교하게 분석합니다.

3. 한국어 금융 태스크 최적화 (SOTA in Korean Finance)
비교: 영어 기반의 FinBERT는 한국어 번역 과정에서 의미 손실이 발생할 수 있고, 기존 한국어 모델들은 금융 뉴스 특유의 '딱딱한 문체'에 취약합니다.

강점: 카카오뱅크에서 실제 금융 서비스 환경을 고려해 공개한 모델인 만큼, 한국어 주식 뉴스 분류 Task에서 가장 높은 신뢰도와 성능을 보입니다.

다만 초기 테스트 결과,

* 금융 뉴스 문맥은 인식하였으나
* **호재 / 악재 / 중립 간 변별력이 충분하지 않음**

이라는 한계가 확인되어 파인튜닝을 진행하게 되었습니다.

---

### 4. 파인튜닝 시행착오 및 개선 과정

총 세 차례의 파인튜닝 실험을 수행하며 데이터 구성에 따른 모델 반응을 관찰했습니다.

#### ① 1차 파인튜닝

* 호재 20,000 / 악재 20,000 / 중립 10,000
* 결과:

  * 전반적인 학습은 이루어졌으나
  * **악재 뉴스에 대한 판별이 충분히 정교하지 않음**

#### ② 1차 추가 파인튜닝

* 기존 모델에 호재/악재 각 약 30,000개 추가 학습
* 결과:

  * 특정 패턴에 과적합
  * **중립 뉴스에 대한 판단 기준 붕괴**

#### ③ 2차 파인튜닝 (최종)

* 학습을 처음부터 다시 진행
* 데이터 비율 재조정:

  * 호재 2,000 / 악재 5,000 / 중립 500
* 결과:

  * 호재와 악재에 대한 **가장 명확한 분리 성능**
  * 실사용 관점에서 가장 안정적인 결과를 보여 최종 모델로 채택

---

### 5. 결과 및 활용

반복적인 파인튜닝과 데이터 비율 조정을 통해,

* 307개 종목 전체 뉴스에 대해
* **일관성 있고 신뢰도 높은 감성 분류 결과**를 생성할 수 있었습니다.

본 모델은 이후 뉴스 기반 종목 분석 및 후속 전략에 활용되었습니다.
